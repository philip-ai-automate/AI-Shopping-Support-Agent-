Step-by-Step Setup Guide
Exactly where to find each credential in Meta's dashboard
⌄
1
Create and verify your Meta Business Account
Go to Meta Business Manager and create an account if you don't have one. Then go to Business Settings → Security Centre and submit your business for verification. You'll need to provide business documents. WhatsApp API access is gated behind this — it typically takes 1–2 business days.
Meta Business Manager ↗ Security Centre ↗
2
Create a Meta Developer App
Go to Meta for Developers and click Create App. When asked for the app type, choose Business. Give it a name (e.g. "YourBusiness WhatsApp") and link it to your Business Account. Once created, click Add Product on the app dashboard and select WhatsApp.
Meta for Developers — My Apps ↗
3
Find your Phone Number ID and WABA ID
In your app dashboard, go to WhatsApp → API Setup. You'll see your test or production phone number listed. Click on the phone number to reveal the Phone Number ID. Just below the number list you'll see the WhatsApp Business Account ID (WABA ID). Copy both — they map directly to the fields in the connect form below.
→ Phone Number ID field → WABA ID field
Open API Setup ↗
4
Generate a permanent System User access token
The default token shown in API Setup expires in 24 hours and will break your integration overnight. To avoid this, go to Meta Business Manager → Business Settings → System Users, create a System User with Admin role, then click Add Assets and grant it access to your WhatsApp App with Manage permission. Finally, click Generate Token, select your app, tick whatsapp_business_messaging and whatsapp_business_management, and copy the token. This token never expires.
→ Access Token field
Business Settings — System Users ↗
5
Copy your App Secret (recommended)
In your Meta Developer App, go to App Settings → Basic. Your App Secret is listed there — click Show to reveal it. Pasting it into PhiXtra lets us verify that webhook messages genuinely came from Meta and reject any forged requests. This is optional but strongly recommended for production.
→ App Secret field
App Settings — Basic ↗
