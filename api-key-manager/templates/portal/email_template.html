Our placeholder tokens look like {{store_name}}, {{cart_value}} etc. If we write them bare in HTML Jinja2 evaluates them — and because store_name / cart_value are not template variables, they become "". Safe approaches used here: • Chip onclick handlers: data-ph="store_name" attribute, JS builds the full {{store_name}} token by concatenating '{{'+'store_name}}' so Jinja2 never sees a complete {{ … }} expression. • Chip label text & info-card text: {% raw %}{{store_name}}{% endraw %} outputs the literal string. • Default template HTML/subject passed via | tojson | safe — the {{…}} tokens are inside a JSON string so Jinja2 ignores them. #} {% block head %} {% endblock %} {% block content %}

ℹ️ How placeholders work

When the email is sent, each placeholder is automatically replaced with the shopper's real data. Click any chip above to insert a placeholder at your cursor.

  • {% raw %}{{store_name}}{% endraw %} — your store's display name
  • {% raw %}{{cart_items}}{% endraw %} — products left in the cart (e.g. Nike Air Max, Blue T-Shirt)
  • {% raw %}{{cart_value}}{% endraw %} — cart total (e.g. £49.99)
  • {% raw %}{{cart_url}}{% endraw %} — direct link to the shopper's cart page
  • {% raw %}{{discount_code}}{% endraw %} — coupon code e.g. COMEBACK10 (only set if Discount Incentive % is configured in Recovery Settings)
  • {% raw %}{{store_url}}{% endraw %} — your store's base URL
